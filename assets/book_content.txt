Chapter 1: Introduction
1.0 The "What is Linux?"
Talk
Okay, let’s be real for a second. Most
people think computers are just these
shiny, expensive boxes made for scrolling

screen, they click the buttons, and that’s
the end of the story. They are basically
living in a world that someone else built
for them, following rules they didn't
create.

But you're reading this because you
aren't "most people." You want to be the
one actually building the world, not just
living in it.

You’ve probably heard the word Linux
before. Maybe you saw it on a forum, or
heard some dev talking about it like it’s a
secret club. Let’s clear the air: Linux isn't
just an "alternative" to Windows or
macOS. It is the secret engine that runs
almost everything on the planet that
actually matters.

Think about it:

    The Android phone in your pocket?
    That’s Linux at its core.
    The massive servers running
    YouTube, Discord, and Instagram? All
    Linux.

    calculating rocket trajectories? Linux.

    Linux (Microkernel, based on
    Linux).

1.1 Why should you actually
care?
Imagine if your computer was a house.

When you use Windows or macOS,
you’re living in a "locked room." It’s a nice
room, sure. It has nice furniture and
pretty paint. But there’s a catch: the
furniture is bolted to the floor. The
windows don't open. And if you want to
repaint the walls, you have to ask a
trillion-dollar company for permission. It’s
a house you paid for, but it’s not really
yours.

Linux is the opposite. Using Linux is like
being handed a giant box of LEGOs, a
high-end toolkit, and the blueprints to the
entire neighborhood. You can move the
walls. You can build a secret basement.
You can change how the electricity flows.
If you don't like how a button looks, you
can literally reach into the code and
change its DNA.

This is what we call Open Source. It
means the "recipe" for the software is
public. Anyone can see it, anyone can
learn from it, and anyone can improve it.
It turns you from a "user" (someone who
just consumes) into a "creator" (someone
who builds).

1.2 The Rebel Story (Briefly)
Back in 1991, a student named Linus
Torvalds was frustrated. Computers were
expensive, and the software was locked
away by big companies. He decided to
write his own operating system kernel just
for fun. He put it on the internet and
basically said, "Hey, I made this. It's free.
If you want to help me make it better, let's
go."

Thousands of people joined him. Why?
Because people love freedom. They
loved the idea of a system built by the
people, for the people. That project
became Linux. Today, it’s the largest
collaborative project in human history.
When you use Linux, you’re joining that
history. You're joining a global community
of hackers, builders, and dreamers who
believe that technology should be open to
everyone.

Using a mouse is like pointing at a picture

faster, it’s more powerful, and—let’s be
honest—it makes you look like a total pro.

Chapter 2: Install (Use yourself)
2.0 The Big Decision
Before we touch a single piece of code,
we have to decide how you’re going to
run Linux. You have three main paths.
Think of this like picking your character
class in an RPG game.

Path A: The Virtual Machine
(The "Simulation")
This is the safest way. You run Linux
inside a window on your current computer
(Windows or Mac). It’s like a "training
room." If you break something, you just
delete the virtual machine and start over.
No harm done to your actual files.

   Pros: 100% safe, no need to restart
   your PC.
   Cons: It’s a bit slower because your
   computer is running two operating
   systems at once.

Path B: The Live USB (The
"Portable OS")
You put Linux on a USB stick. You plug it
in, tell your computer to "boot" from the
USB, and boom—you’re in Linux. When
you unplug it and restart, you’re back in
Windows.

   Pros: You can carry your entire OS in
   your pocket and use it on any
   computer.
   Cons: Unless you set up
   "persistence," your files won't save
   when you unplug.
Path C: Bare Metal (The "Main
Character" Energy)
This is "Dual Booting" or replacing your
OS entirely. You install Linux directly onto
your hard drive. This is how the pros do
it.

      Pros: Absolute maximum speed and
      performance.
      Cons: If you mess up the installation,
      you might lose your schoolwork or
      those Minecraft screenshots you care
      about. Always back up your files
      first.




2.1 The Gear You Need
To get started, you’re going to need a few
things in your digital inventory:

  1. An ISO File: This is a "disc image."
     It’s basically the entire Linux OS
      packed into one big file. (I
   recommend starting with something
   like Ubuntu or Linux Mint because
   they just work.)
 2. A USB Drive: At least 8GB. Make
   sure there’s nothing important on it,
   because we are going to wipe it
   clean.
 3. BalenaEtcher or Rufus: These are
    the "magic wands" that take that ISO
   file and burn it onto your USB so it
   can actually boot.




2.2 Making the Magic Key
(Flashing the USB)
Once you have your ISO (Ubuntu or
Pop!_OS is good) and your tool (let's use
BalenaEtcher because it's super simple),
follow these steps:

 1. Plug in your USB.
 2. Open BalenaEtcher.
 3. Select "Flash from file" and pick
    your Linux ISO.
 4. Select Target: Pick your USB drive
    (triple-check you didn't pick your
    external hard drive by mistake!).
 5. Hit "Flash!" and wait for the bar to
    finish.

Congratulations. You now hold the key to
a whole new world in your hand.




2.3 Entering the BIOS (The
Boss Fight)
This is the part that scares people, but it’s
actually easy once you know the trick. To
make your computer load Linux instead
of Windows, you have to talk to the BIOS
(the "brain" of your motherboard).

 1. Restart your computer.
 2. Spam the "Magic Key": As soon as
    the screen turns black, keep tapping
    the key that opens the Boot Menu. It’s
    usually F12 , F2 , F10 , or Delete .
    (You might have to Google your
    laptop model to find the right one).
 3. Select your USB: Use the arrow
    keys to find your USB drive and hit
    Enter.

If you see a logo that isn't Windows,
you’ve done it. You’re officially "in."




2.4 The Actual Install
Once the Linux desktop loads, you’ll see
an icon that says "Install." Double-click
it.
The installer will ask you a few questions:
- Language: Keep it simple.
- Keyboard Layout: Usually "English
(US)".
- Updates: Check the box that says
"Install third-party software for graphics
and Wi-Fi." Trust me, it saves a lot of
headaches later.
The Scary Screen:
Partitioning
The installer will ask where to put Linux.
- If you want to keep Windows, choose
"Install alongside Windows."
- If you want to go full-pro, choose
"Erase disk and install Linux."
(Warning: This deletes EVERYTHING on
that drive).

Set your Username (make it something
cool) and a Password.
PRO TIP: Do NOT forget your password.
In Linux, if you lose your password, you
can’t just click "Forgot Password" to get a
reset link in your email. You are the boss
here, and the boss has to remember their
keys.




2.5 The Final Step
Once the progress bar finishes, it will tell
you to restart and "Remove installation
medium." Pull the USB out and hit Enter.

When your computer boots up and you
see that fresh Linux login screen... take a
breath. You just successfully escaped the
"locked room."

Welcome to the club.

Chapter 4: The magical box
4.0 Facing the Void
Alright, it’s time. Open your terminal using
Ctrl + Alt + T .

What do you see? Probably a black or
dark purple window with some text that
looks like yourname@computername:~$
and a little blinking rectangle.

Most people see this and think it’s a
broken computer from 1985. They think,
"Where are my icons? Where is the 'Start'
button? How am I supposed to do
anything?" Welcome to the CLI
(Command Line Interface). It’s called the
"Magical Box" for a reason. In the GUI
(the desktop with icons), you are a
passenger. In the CLI, you are the pilot.
You aren't just clicking on things; you are
talking directly to the computer’s soul.
4.1 The Prompt:
Deciphering the Code
Before we type anything, let’s look at that
weird line of text: anagh@antheros:~$

It’s not just random gibberish. It’s actually
giving you a status report:

 1. anagh: That’s you (the User).
 2. @: Just a separator meaning "at."
 3. antheros: That’s the name of your
    computer (the Hostname).
 4. ~: This is a very important symbol
    called the Tilde. In Linux, it means
    "Home." It tells you that you are
    currently hanging out in your user's
    home folder.
 5. $: This means you are a "standard
    user." If you see a # instead, it
    means you have "Root" powers (more
    on that later—be careful with # !).




4.2 Why Use the Box?
You might be thinking, "Why would I type
commands when I can just click a
folder?"

Here’s why the pros use the terminal:

   Speed: I can create 100 folders in 0.5
   seconds using one line of text. Try
   doing that with a mouse without
   getting a cramp in your hand.
   Automation: You can write a "script"
   (a list of commands) and tell the
   computer to run it every day at 3:00
   AM.
   Remote Control: Most servers in the
   world don't even have a screen. You
   have to use a terminal to talk to them
   over the internet.
   Fixing things: If your desktop
   environment crashes and the screen
   goes frozen, the terminal is often the
   only way to get back in and fix the
   mess.
4.3 Your First Command:
whoami
Let’s try something simple. Type this and
hit Enter:
whoami

The computer should reply with your
username. It’s a basic command, but it
proves the computer is listening. You
asked a question, and it answered.




4.4 The "Shell": The Ghost
in the Machine
The terminal is just the window. The thing
actually processing your commands is
called the Shell.

The most popular shell (and the one you
are likely using) is called Bash (Born
Again Shell). There are others like Zsh or
Fish, but Bash is the industry standard.

Think of the Shell as a translator.
 1. You type whoami .
 2. The Shell looks through its
   "dictionary" to see what that means.
 3. It finds the program called whoami .
 4. It tells the CPU to run it.
 5. It shows you the result.




4.5 Managing the Chaos:
clear
Sometimes the terminal gets cluttered
with too much text and it starts looking
messy.
Type:
clear

Poof. Everything disappears and you’re
back to a clean slate.




4.6 The "Super User" (The
Boss Mode)
In Linux, there are things a normal user
isn't allowed to do—like deleting system
files or installing new software. To do
those things, you need to use sudo .

sudo stands for "Super User DO."
When you put sudo before a command,
you’re basically saying, "I am the boss,
do what I say." The computer will ask for
your password. When you type it, you
won't see any stars or dots—the screen
will stay blank. Don't worry, it’s still
recording your typing. Just hit Enter when
you're done.

Warning: Using sudo is like holding a
loaded lightsaber. It’s powerful, but if you
swing it the wrong way, you can cut your
own legs off (or delete your entire OS).
Use it only when you have to.

Chapter 5: Basic Spells
5.0 The Art of Navigation
In the "Normal World" (Windows or
macOS), you find your files by clicking on
folders and watching windows pop up. In
Linux, you don't need windows. You
move through the system like a ghost.

But to do that, you need to know where
you are. Remember that little ~ (Tilde)
we talked about? That’s your starting
point. But how do you see what’s actually
inside your current location?
5.1 Spell #1: ls (The X-
Ray Vision)
The first spell every Linux user learns is
ls . It stands for List.

Type it and hit Enter:
ls

You’ll see a list of everything in your
current folder. Usually, it’s things like
Documents , Downloads , and Desktop .


The "Pro" Versions of ls :
Standard ls is cool, but sometimes
things are hidden. In Linux, any file or
folder that starts with a dot . is invisible
to the normal ls command.

     ls -a : Shows all files, including the
     hidden ones.
     ls -l : Shows the long list. This
     gives you extra info like how big the
     file is and when it was made.
     ls -lh : The "Human" version.
     Instead of showing file sizes in weird
    bytes, it shows them in KB, MB, or
    GB.




5.2 Spell #2: pwd (The
GPS)
Sometimes you go so deep into folders
that you forget where you started. That’s
where pwd comes in. It stands for Print
Working Directory.

Type:
pwd

It will spit out your exact "Global
Address," like
/home/anagh/Documents/Projects . It’s
the "You Are Here" map for your terminal.




5.3 Spell #3: cd (The
Teleportation Spell)
Now that you can see folders, how do
you go inside them? Use cd , which
stands for Change Directory.

    To go into a folder: cd Documents
    To go back one step: cd .. (This is
    super important! The two dots mean
    "the folder above this one").
    To teleport straight home: Just type
    cd by itself and hit Enter. No matter
    how lost you are, this brings you back
    to ~ .

Pro Tip: You don't have to type the whole
folder name. Type the first two letters and
hit the Tab key. The terminal will finish the
word for you. If it doesn't, it means there’s
more than one folder starting with those
letters—hit Tab twice to see the list!




5.4 Spell #4: man (The
Instruction Manual)
You don't have to memorize everything.
Every command in Linux comes with its
own manual.

Type:
man ls

This opens a book inside your terminal
that explains every single thing ls can
do. To exit the manual and get back to
your terminal, just press q .




5.5 Spell #5: history (The
Time Machine)
Ever typed a really long, cool command
and then forgot what it was five minutes
later?
Type:
history

Linux keeps a log of every single
command you’ve typed in this session.
You can scroll up to see what you did.
Life Hack: Instead of typing history ,
you can just press the Up Arrow on your
keyboard to cycle through your last few
commands. It’s a massive time-saver.




5.6 Putting it Together: The
Mini-Quest
Let’s see if you’ve mastered the basics.
Try to do this sequence without looking at
the notes:

 1. Check where you are right now
    ( pwd ).
 2. List everything in that spot ( ls ).
 3. Teleport into your Downloads folder
    ( cd Downloads ).
 4. Look for any hidden files ( ls -a ).
 5. Teleport back to your Home folder
    using only two characters ( cd ).

If you did that without any errors, you’ve
officially finished the "Movement" tutorial.
You’re no longer stuck in one spot—you
own the whole drive.

Chapter 6: Creation Spells
6.0 From Ghost to Creator
In the last chapter, you learned how to
move through the system like a ghost.
You could see everything, but you
couldn't touch or change anything. That
changes today.

In Linux, creating things isn't about right-
clicking and hitting "New Folder." It’s
about using specific commands that act
like building blocks. Once you master
these, you can build entire file systems in
seconds.




6.1 Spell #1: mkdir (The
Architect Spell)
If you want to create a new "room" (a
folder/directory), you use mkdir . It
stands for Make Directory.

The Basics:
Type:
mkdir MySecretBase

Now type ls . You’ll see a brand-new
folder called MySecretBase .
The Pro Move (The -p flag):
What if you want to create a folder inside
a folder inside another folder? Instead of
doing it one by one, you can do this:
mkdir -p Level1/Level2/Level3

The -p stands for "parents." It tells
Linux: "If the parent folders don't exist
yet, just build them for me."




6.2 Spell #2: touch (The
Magic Wand)
To create a file out of thin air, you use
 touch .

Type:
touch notes.txt

If you check your folder now, you have a
file named notes.txt . It’s empty for
now, but it exists.

Wait, why is it called "touch"?
In Linux, every file has a timestamp
(when it was last touched). If the file
already exists and you run touch on it, it
doesn't delete it—it just updates the "Last
Seen" time to right now. But if the file
doesn't exist, Linux creates it.




6.3 Spell #3: cp (The
Cloning Spell)
Need a backup? Or want to move a copy
of a file somewhere else? Use cp
(Copy).

How it works: cp [Source]
[Destination]

Try this:
cp notes.txt backup_notes.txt

Now you have two identical files.

Copying Folders:
If you try to copy a folder with cp , Linux
will complain. You have to tell it to be
"Recursive" (copy the folder and
everything inside it).
cp -r MySecretBase BackupBase




6.4 Spell #4: mv
(Teleportation & Re-
branding)
The mv command stands for Move. It
does two very different things depending
on how you use it.

1. Moving a file:
mv notes.txt Documents/
This takes notes.txt and puts it inside
your Documents folder.

2. Renaming a file:
In Linux, there is no "Rename" command.
To rename something, you just "move" it
from the old name to the new name.
mv notes.txt journal.txt
Poof. notes.txt is gone, and
journal.txt is in its place.
6.5 Speedrun Challenge:
The Instant Lab
Let’s see how fast you are. Try to do this
entire sequence in under 30 seconds:

 1. Make a folder called Lab .
 2. Go inside Lab .
 3. Create three files: test1.txt ,
    test2.txt , and test3.txt . (Tip:
    You can do touch test1.txt
    test2.txt test3.txt all in one
    line!)
 4. Rename test1.txt to
    experiment.txt .
 5. Move test2.txt back to your Home
    folder ( mv test2.txt .. ).

If you did that, you just managed five files
and folders without ever touching your
mouse. That’s the power of being a
creator in Linux.

   You can build deep folder structures
   with mkdir -p .
   You can summon empty files with
    touch .
   You know that mv is for both moving
   and renaming.
   You can clone folders using the -r

Chapter 7: Destroying Spells
7.0 The Point of No Return
In the last chapter, you learned how to
build a digital empire. But every architect
needs to know how to tear down the old
stuff to make room for the new.
In Windows or macOS, when you delete
a file, it goes to a "Recycle Bin" or
"Trash." It sits there waiting for you to
change your mind.

In the Linux Terminal, there is no
Trash.

When you run a destroying spell, the file
is wiped from the face of the earth
instantly. There is no "Undo" button.
There is no "Are you sure?" pop-up. If
you tell Linux to delete your entire
homework folder, it will do it without a
single tear.




7.1 Spell #1: rm (The
Eraser)
The most basic destroying spell is rm . It
stands for Remove.

To delete a single file:
rm notes.txt
The file is gone. If you type ls , you
won’t see it anymore.

To delete multiple files at once:
rm file1.txt file2.txt file3.txt




7.2 Spell #2: rmdir (The
Empty Room Destroyer)
If you want to delete a folder, you might
try rm , but Linux will stop you and say:
"rm: cannot remove 'MyFolder': Is a
directory."

If the folder is completely empty, you
can use rmdir :
rmdir MyEmptyFolder

If there is even one tiny hidden file inside
that folder, rmdir will fail. It’s a "safety"
command to make sure you don't
accidentally delete a folder full of stuff.
7.3 Spell #3: rm -r (The
Demolition Ball)
What if you want to delete a folder and
everything inside it? You use the
Recursive flag ( -r ).

rm -r MyBigProject

This tells Linux: "Go into the folder, delete

11.1 The "Shebang" (The
Secret Header)
Every Bash script needs to start with a
very specific first line. It’s called the
Shebang.

#!/bin/bash

This tells your computer: "Hey, don't just
treat this like a random text file. Use the
Bash shell to read and execute these
instructions." Without this line, the
computer might get confused about which
language you’re speaking.




11.2 Your First Script:
 hello.sh
Let's use the EOF trick from Chapter 9 to
build a script instantly:


  cat <<EOF > hello.sh
  #!/bin/bash
  echo "What is your name?"
  read USER_NAME
  echo "Hello \$USER_NAME! You
  are currently using
  \$(hostname)."
  echo "Today is \$(date)"
  EOF


Breaking it down:
read : This command pauses the script
and waits for you to type something. It
saves whatever you type into a variable
called USER_NAME.

$USER_NAME : The $ tells Bash to go look
inside the variable and print the value.

$(command) : This tells Bash to run a
command (like date) and put the result
right there in the sentence.

11.3 The "Permission" Wall
If you try to run your script by typing
./hello.sh , Linux will probably slap you
with an error: " Permission denied. "

Why? Because by default, Linux doesn't
trust new files. It thinks: "I don't know if
this is a safe program or just a text file, so
I'm not going to let it run."

You have to grant the "Execution" power
using chmod (Change Mode):
chmod +x hello.sh

Now, you can run it:
./hello.sh

chmod +x clean.sh```

Chapter 17: The Linux Gaming Revolution
Linux is no longer the "work-only" OS.
With the rise of the Steam Deck and the
optimization of the Linux Kernel, it has
become a powerhouse for competitive
gaming. For a sniper in Black Ops 3,
every millisecond of input lag matters.

1. Proton and the Magic of
WINE
Most games are written for Windows
(DirectX). Linux uses Vulkan. Proton is
the translation layer that acts as a real-
time interpreter.

    WINE (Wine Is Not an Emulator): It
    creates a "prefix" (a fake Windows C:
    drive) where your game can live.
    Steam Play: Go to Settings >
    Compatibility to enable Proton for all
    titles.

2. Tuning for Zero Latency
To get the best performance for FPS
gaming, you need to prioritize your
hardware:

   GameMode: A daemon developed by
   Feral Interactive. Run gamemoderun
    ./game_executable to stop
   background processes from stealing
   your CPU cycles.
   Custom Kernels: Using a "Zen" or
   "Liquorix" kernel can provide better
   "scheduling," meaning the OS
   responds faster to your mouse clicks.

3. The Boltz-DE Philosophy
A gaming desktop environment shouldn't
have "bloat." By stripping away
transparency effects and heavy
animations, you save RAM for the game
itself.

18.4 Cron: The Invisible
Alarm Clock
The Crontab is the most powerful
automation tool in Linux. It is a
background service that watches the
clock and runs your scripts at a specific
time, even while you are asleep.

To edit your schedule, type: crontab -e

The format looks like a secret code: * *
* * * /path/to/script.sh

 1. Minute (0-59)
 2. Hour (0-23)
 3. Day of Month (1-31)
 4. Month (1-12)
 5. Day of Week (0-6)

The "Sunday Backup"
Spell:
0 0 * * 0
/home/anagh/scripts/backup.sh This
tells Linux: "Every Sunday (0) at Midnight
(0:00), run my backup script."

The Uncomplicated Firewall (UFW) is
your first line of defense.

    sudo ufw default deny incoming
    sudo ufw default allow outgoing
   This blocks everyone from trying to
   enter your computer while letting you
   browse the web freely.

If someone steals your laptop, they
shouldn't be able to see your code. LUKS
encrypts your entire hard drive. Without
your "master key" (passphrase), the data
looks like random noise.

3. Privacy vs. Convenience
Linux gives you the choice to say "No" to
telemetry. While Windows and macOS
send "usage data" back to HQ, your
Linux system only talks to who you tell it
to.

Chapter 21: Digital Sovereignty & Permissions
To be a true CEO of your system, you
must control who can touch your files. We
use chmod to set permissions.

The chmod (change mode) command in
Unix-like operating systems modifies file
or directory permissions, controlling read
( r ), write ( w ), and execute ( x ) access
for owners, groups, and others. It uses
octal numbers (e.g., 755 ) or symbolic
notation (e.g., u+x ) to set permissions,
supporting recursive changes via the -
R flag.

Key chmod Concepts & Syntax

    Syntax: chmod [options] mode
    file_name .
    Symbolic Modes: Uses letters—
    u (user), g (group), o (others), a (all)
    —and operators—+ (add), -
    (remove), = (set exactly).
          Example: chmod u+x
          file (Adds execute permission
          for the user).
  Numeric (Octal) Modes: Three digits
  represent user, group, and others
  (e.g., chmod 755 file ).
       4: Read ( r )
       2: Write ( w )
       1: Execute ( x )
       0: No permission
  Common Examples:
       chmod 755 file : Owner can
       read/write/execute; group/others
       can read/execute.
       chmod 644 file : Owner can
       read/write; group/others can
       read.
       chmod 700 file : Owner has
       full access; others have none.
       chmod +x script.sh : Makes a
       file executable for all.
       chmod -R 755 directory :
       Recursively changes
       permissions for a directory and
       its contents.

Common Options
    -R : Changes files and directories
    recursively.
    -v : Outputs a diagnostic for every
    file processed.
    -f : Suppresses error messages.

Understanding Permissions

    Read (r): View file contents or list
    directory contents.
    Write (w): Modify/delete files or
    create/delete files in a directory.
    Execute (x): Run a file as a program
    or enter a directory.

📍 Finding Your Way
    pwd – Print Working Directory:
   Shows exactly where you are (your

    mkdir [name] – Make Directory:
   Create a new folder.

    cp [file] [new] – Copy: Clone a
   file. (Use -r for folders).
    mv [old] [new] – Move/Rename:
   Move a file or change its name.
    clear – Clear: Wipe the terminal
   screen forma fresh start.

rm [file] –

Remove: Delete a file
permanently. (No Trash
Bin!)

    rm -r [folder] – Recursive
   Remove: Delete a folder and
   everything inside it.
    rmdir [folder] – Remove
   Directory: Delete an empty folder.

  sudo – SuperUser Do: Run any
